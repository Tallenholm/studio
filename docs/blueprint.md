# **App Name**: Fleet Check

## Core Features:

- Pre-Trip Checklist: Provide a checklist for pre-trip inspection, including truck, trailer, and skid steer.
- Post-Trip Checklist: Provide a checklist for post-trip inspection, including truck, trailer, and skid steer.
- AI Anomaly Detection: Use generative AI to analyze the pre-trip inspection reports to detect anomalies based on past reports. This tool determines whether anomalies merit mechanic intervention, or if a report may require changes to procedure.
- Manual VIN Entry: Enable the user to manually enter the Vehicle Identification Number of all equipment. This is necessary to provide specific instruction regarding inspection of the identified vehicle or tool.
- Status Recording: Record inspection status (pass/fail) for each item on the checklist.
- Inspection Instructions: Provide clear instructions/criteria for each inspection item.
- Report Generation/Export: Generate a report based on inspection data. Provide an option to save this report for future inspections.

## Style Guidelines:

- Primary color: Deep sky blue (#00BFFF) for a sense of reliability and safety.
- Background color: Light gray (#E0E0E0) to provide a neutral backdrop.
- Accent color: Orange (#FFA500) to draw attention to important elements like alerts or critical issues.
- Body and headline font: 'PT Sans', sans-serif.
- Use clear, simple icons to represent each inspection item. These icons should be easily understandable to facilitate quick assessment.
- The layout should be intuitive and straightforward. The checklist should be well-organized for easy navigation.
- Subtle transitions when marking items as 'pass' or 'fail'. A simple loading animation when generating reports.
